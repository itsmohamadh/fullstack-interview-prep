# Behavioral Interview Prep

> "Your technical skills got you the interview. Your communication gets you the offer."

## The STAR Framework

**STAR** is the gold standard for behavioral answers. Structure every story this way:

| Letter | Meaning       | What to include                                                    | Time   |
| ------ | ------------- | ------------------------------------------------------------------ | ------ |
| **S**  | **Situation** | Set the context. Where/when? What was the team? What was at stake? | 10-15% |
| **T**  | **Task**      | What needed to be done? What was your responsibility?              | 10-15% |
| **A**  | **Action**    | What did **YOU** do? Use "I" not "we". Include technical details.  | 50-60% |
| **R**  | **Result**    | What happened? **Quantify** if possible. What did you learn?       | 15-20% |

### Example STAR Answer

**Q: "Tell me about a time you fixed a critical bug in production."**

> **S:** "At my previous company, we had an e-commerce platform handling ~10,000 daily orders. One Friday afternoon, our payment processing started failing for about 5% of transactions."
>
> **T:** "I was the on-call engineer and needed to identify the root cause and restore service before the weekend rush."
>
> **A:** "I immediately checked the error logs and noticed a timeout pattern. I traced it to a third-party payment API that had changed their rate limits. I implemented a retry mechanism with exponential backoff, added circuit breaker patterns, and set up monitoring alerts for API latency."
>
> **R:** "We restored service within 45 minutes. The retry mechanism recovered 98% of failed transactions. I documented the incident and the API provider's changes so the team could update our integration spec. The monitoring alerts caught similar issues twice more before they became incidents."

## The 5 Stories Rule

**Prepare 5-7 core stories** that can flex to answer 80% of behavioral questions:

| Story Type                           | Questions it answers                                                                 |
| ------------------------------------ | ------------------------------------------------------------------------------------ |
| **1. A Success Story**               | "Project you're proud of", "Technical decision you made", "Went above and beyond"    |
| **2. A Failure Story**               | "Mistake you made", "What would you do differently", "Learned from experience"       |
| **3. A Conflict Story**              | "Disagreed with teammate", "Handled difficult person", "Pushed back on requirements" |
| **4. A Leadership/Mentorship Story** | "Mentored someone", "Led a project", "Influenced without authority"                  |
| **5. A Tight Deadline Story**        | "Pressure situation", "Ambiguous requirements", "Multiple priorities"                |
| **6. A Technical Deep Dive**         | "Technical challenge", "Complex system you built", "Optimization you made"           |
| **7. A Customer/User Focus Story**   | "User empathy", "Accessibility", "UX decision"                                       |

**Pro tip:** Write each story in STAR format. Practice them until they feel natural, not rehearsed. You should be able to tell any story in 90-120 seconds.
