## Core SQL

[ ] Can explain SQL execution order
[ ] Can explain WHERE vs HAVING
[ ] Can write INNER and LEFT JOINs
[ ] Can find duplicate records
[ ] Understand GROUP BY and aggregates
[ ] Know COUNT(*) vs COUNT(column)


## Database Design

[ ] Understand primary and foreign keys
[ ] Can design one-to-many relationships
[ ] Understand normalization
[ ] Know when denormalization makes sense


## Performance

[ ] Understand indexes
[ ] Know when indexes help and hurt
[ ] Can explain how to debug a slow query
[ ] Understand EXPLAIN at a high level
[ ] Know OFFSET vs cursor pagination


## Reliability

[ ] Understand transactions
[ ] Can explain ACID
[ ] Know DELETE vs TRUNCATE vs DROP


## Security

[ ] Can explain SQL injection
[ ] Know parameterized queries
[ ] Understand why ORMs help


## Real World

[ ] Have a story for the most complex query you wrote
[ ] Can explain a database feature you built at the dealership project
[ ] Can discuss tradeoffs instead of just syntax


If you can confidently explain ~80% of this list,
you are already at a strong mid-level and many
